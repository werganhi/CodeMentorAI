# Дипломная документация

## Тема

**Разработка мобильного приложения для обучения программированию с элементами искусственного интеллекта**

## Аннотация

В дипломной работе рассматривается разработка мобильного приложения CodeMentor AI, предназначенного для обучения начинающих пользователей основам программирования. Приложение объединяет короткие теоретические уроки, примеры кода, тестовые задания, практический тренажер, справочник терминов и ИИ-наставника. В демонстрационной версии ИИ-наставник реализован локально, что позволяет запускать приложение без подключения к серверу и API-ключей. Архитектура проекта предусматривает дальнейшую интеграцию внешней языковой модели.

## Введение

Навыки программирования становятся важной частью современного образования. При этом начинающим пользователям часто сложно самостоятельно разобраться в синтаксисе, логике алгоритмов и сообщениях об ошибках. Традиционные учебники дают теорию, но не всегда обеспечивают интерактивную практику и быструю обратную связь.

Мобильное приложение для обучения программированию решает эту проблему за счет доступности, коротких учебных блоков и постоянного взаимодействия с пользователем. Добавление элементов искусственного интеллекта повышает качество поддержки: студент может получить объяснение ошибки, рекомендацию по следующей теме и пример практического задания.

## Актуальность

Актуальность темы обусловлена следующими факторами:

- ростом спроса на цифровые навыки;
- популярностью мобильного формата обучения;
- сложностью самостоятельного изучения программирования;
- развитием ИИ-инструментов, которые могут выполнять роль учебного наставника;
- необходимостью персонализированного обучения.

Мобильное приложение CodeMentor AI ориентировано на практическое освоение базовых тем программирования. Пользователь получает не только теорию, но и задания, проверку ответов, справочник и подсказки наставника.

## Объект и предмет исследования

**Объект исследования:** процесс обучения основам программирования.

**Предмет исследования:** мобильное приложение с интерактивными учебными модулями и элементами искусственного интеллекта.

## Анализ предметной области

Обучение программированию включает несколько ключевых этапов:

- понимание базовых команд;
- изучение переменных и типов данных;
- освоение условий и циклов;
- работа с функциями;
- работа со структурами данных;
- отладка ошибок;
- применение знаний в мини-проектах.

Сложность заключается в том, что программирование требует не только запоминания синтаксиса, но и развития алгоритмического мышления. Поэтому учебное приложение должно сочетать объяснение, пример, практику и обратную связь.

## Анализ аналогов

На рынке существуют приложения для обучения программированию: SoloLearn, Mimo, Programming Hub, Stepik и другие. Они предлагают уроки и задания, однако часто имеют ограничения:

- часть функций доступна только по подписке;
- задания не всегда адаптированы под конкретного пользователя;
- объяснение ошибок может быть недостаточно подробным;
- не во всех приложениях есть полноценный ИИ-наставник;
- интерфейс может быть перегружен для начинающего пользователя.

Разрабатываемое приложение ориентировано на простую структуру, локальный запуск и понятный демонстрационный прототип для дипломной защиты.

## Выбор технологий

Для реализации выбран Flutter, так как он позволяет разрабатывать мобильные приложения с единой кодовой базой для Android и iOS. Flutter предоставляет готовые компоненты Material Design, высокую скорость разработки и хорошую производительность интерфейса.

Основные технологии:

- **Flutter** - разработка мобильного интерфейса;
- **Dart** - язык программирования приложения;
- **Material 3** - современная система компонентов;
- **shared_preferences** - локальное хранение прогресса.

## Архитектура приложения

Приложение построено как автономный Flutter-проект. Основная логика находится в файле `lib/app.dart`, точка входа - `lib/main.dart`.

Структура приложения включает следующие модули:

- главный экран;
- модуль уроков;
- экран подробного урока;
- модуль практики;
- справочник терминов;
- ИИ-наставник;
- профиль и локальный прогресс.

Данные уроков, терминов и практических заданий хранятся локально в виде Dart-моделей. Такой подход упрощает запуск прототипа и демонстрацию на защите.

## Функциональность приложения

### Главный экран

Главный экран отображает общую картину обучения:

- прогресс курса;
- количество XP;
- текущий стрик;
- число завершенных уроков;
- быстрые действия;
- следующий рекомендуемый урок.

Главный экран помогает пользователю быстро понять, где он находится в обучении и что делать дальше.

### Уроки

Экран уроков содержит список тем по программированию. В приложении представлены уроки:

- старт в Python;
- переменные и типы;
- условия и логика;
- циклы;
- функции;
- списки и словари;
- отладка и ошибки;
- ИИ в приложениях.

Каждый урок имеет подробный экран с вкладками:

- **Теория** - объяснение темы;
- **Код** - примеры и термины;
- **Тест** - проверка понимания.

После прохождения теста урок отмечается завершенным, а пользователю начисляется XP.

### Практика

Практический модуль содержит небольшие задания на чтение кода и прогнозирование результата. Пользователь вводит ответ, приложение сравнивает его с ожидаемым значением и показывает результат. При правильном ответе начисляется дополнительный опыт.

Такой формат тренирует важный навык программиста: умение читать код и понимать, что он выполнит.

### Справочник

Справочник содержит термины программирования, сгруппированные по категориям. Пользователь может искать по названию, описанию и синтаксису. Справочник помогает быстро вспомнить назначение конструкции и пример использования.

### ИИ-наставник

ИИ-наставник представлен отдельным экраном диалога. В прототипе он работает локально: анализирует ключевые слова в запросе и возвращает заранее подготовленные учебные ответы. Например, наставник может объяснить:

- ошибку `NameError`;
- принцип работы цикла `for`;
- условные операторы;
- стратегию изучения Python;
- пример практического задания.

Такое решение позволяет показать идею интеллектуальной поддержки без необходимости подключать внешний сервис.

## Элементы искусственного интеллекта

В приложении реализованы следующие ИИ-элементы:

1. **ИИ-наставник** - экран диалога с учебными ответами.
2. **Рекомендательная логика** - главный экран показывает следующий урок на основе прогресса.
3. **Подсказки для самопроверки** - в модуле практики пользователь получает шаблоны запросов к ИИ.
4. **Объяснение ошибок** - наставник объясняет типовые ошибки начинающих разработчиков.

В промышленной версии локальная логика может быть заменена запросами к языковой модели через API. Для этого можно добавить сервисный слой, который отправляет промпт пользователя на сервер и получает ответ модели.

## Пользовательский сценарий

Типовой сценарий работы:

1. Пользователь открывает приложение.
2. На главном экране видит прогресс и следующий урок.
3. Переходит в раздел уроков.
4. Изучает теорию и примеры кода.
5. Проходит тест.
6. Получает XP и завершает урок.
7. Переходит в практику и решает мини-задание.
8. При затруднении открывает ИИ-наставника.
9. Использует справочник для уточнения терминов.
10. Возвращается к обучению на следующий день, сохраняя стрик.

- Главная;

## Локальное хранение данных

Для хранения прогресса используется пакет `shared_preferences`. Сохраняются:

- список завершенных уроков;
- XP;
- стрик;
- дата последнего входа;
- имя пользователя.

Это позволяет приложению работать автономно и сохранять состояние после перезапуска.

## Безопасность и ограничения

Приложение не хранит персональные данные на сервере. Все данные сохраняются локально на устройстве. В прототипе нет регистрации и сетевого обмена, что снижает риски утечки данных.

Ограничения прототипа:

- ИИ-наставник не подключен к реальной языковой модели;
- практические задания не выполняют код в песочнице;
- прогресс хранится только на одном устройстве;
- нет облачной синхронизации.
