# Техническое задание

## 1. Наименование проекта

Мобильное приложение **CodeMentor AI** для обучения программированию с элементами искусственного интеллекта.

## 2. Основание для разработки

Разработка выполняется в рамках дипломной работы на тему: **"Разработка мобильного приложения для обучения программированию с элементами искусственного интеллекта"**.

## 3. Цель разработки

Создать мобильное приложение, которое помогает начинающим пользователям изучать основы программирования через короткие уроки, практические задания, справочник терминов, систему прогресса и ИИ-наставника.

## 4. Назначение системы

Приложение предназначено для самостоятельного обучения программированию на базовом уровне. Система должна:

- объяснять основные темы программирования простым языком;
- показывать примеры кода;
- закреплять материал через тесты и мини-задания;
- хранить индивидуальный прогресс пользователя;
- имитировать работу ИИ-наставника для объяснения ошибок и подбора рекомендаций;
- быть удобной для использования на мобильном устройстве.

## 5. Целевая аудитория

Основная аудитория:

- школьники и студенты, начинающие изучать программирование;
- пользователи без опыта разработки;
- обучающиеся, которым нужен быстрый справочник по базовым понятиям;
- преподаватели, которым нужен демонстрационный мобильный прототип.

## 6. Функциональные требования

### 6.1 Главный экран

Главный экран должен отображать:

- название приложения;
- приветствие пользователя;
- общий прогресс курса;
- количество XP;
- текущий стрик обучения;
- количество завершенных уроков;
- быстрый переход к урокам, практике, справочнику и ИИ-наставнику.

### 6.2 Модуль уроков

Модуль уроков должен обеспечивать:

- отображение списка уроков;
- фильтрацию по направлениям: основы, алгоритмы, архитектура, данные, качество, ИИ;
- открытие подробного экрана урока;
- отображение теории;
- отображение примеров кода;
- отображение ключевых терминов;
- прохождение теста по уроку;
- начисление XP за завершение урока.

### 6.3 Модуль практики

Модуль практики должен обеспечивать:

- показ коротких заданий по чтению кода;
- ввод ответа пользователем;
- проверку ответа;
- вывод подсказки;
- начисление XP за успешное выполнение;
- переход к следующему заданию.

### 6.4 Справочник

Справочник должен обеспечивать:

- отображение терминов программирования;
- поиск по названию, описанию и синтаксису;
- фильтрацию по категориям;
- отображение описания, синтаксиса и примера.

### 6.5 ИИ-наставник

ИИ-наставник должен обеспечивать:

- экран диалога;
- ввод пользовательского вопроса;
- быстрые шаблоны запросов;
- генерацию учебного ответа на основе ключевых слов;
- объяснение типовых ошибок, условий, циклов и стратегии обучения.

В прототипе ИИ реализован локально без подключения к внешнему API. Архитектура допускает дальнейшее подключение языковой модели через REST API.

### 6.6 Профиль и прогресс

Система должна хранить:

- имя пользователя;
- количество XP;
- завершенные уроки;
- текущий стрик;
- дату последнего входа.

Хранение выполняется локально через `shared_preferences`.

## 7. Нефункциональные требования

### 7.1 Удобство интерфейса

Интерфейс должен быть:

- адаптирован под мобильные экраны;
- понятным для начинающих;
- визуально современным;
- построенным на нижней навигации;
- без перегруженных экранов.

### 7.2 Надежность

Приложение должно:

- запускаться без подключения к серверу;
- сохранять прогресс локально;
- корректно обрабатывать пустой ввод в практических заданиях и ИИ-чате;
- не требовать регистрации для демонстрации.

### 7.3 Производительность

Приложение должно:

- быстро открывать основные экраны;
- использовать локальные данные;
- не выполнять тяжелых сетевых запросов в прототипе.

### 7.4 Масштабируемость

Архитектура должна позволять:

- добавлять новые уроки;
- расширять базу терминов;
- подключить реальный API ИИ;
- добавить авторизацию и облачное хранение прогресса.

## 8. Технологические требования

Технологический стек:

- Flutter;
- Dart;
- Material 3;
- `shared_preferences` для локального хранения.

Платформы:

- Android;
- iOS при наличии настроенного окружения;
- Web/Desktop как дополнительные цели Flutter.

## 9. Требования к дизайну

Дизайн должен использовать:

- чистую карточную структуру;
- контрастные акцентные цвета;
- иконки Material;
- понятные кнопки действий;
- читаемые блоки кода;
- темную и светлую системную тему.

## 10. Состав экранов

В приложении должны быть реализованы:

- главный экран;
- список уроков;
- экран урока с вкладками "Теория", "Код", "Тест";
- экран практики;
- справочник терминов;
- экран ИИ-наставника;
- модальное окно профиля.

## 11. Критерии приемки

Проект считается выполненным, если:

- приложение запускается как Flutter-проект;
- главный экран отображает прогресс;
- уроки открываются и содержат теорию, код и тест;
- после прохождения теста урок отмечается завершенным;
- практический тренажер проверяет ответы;
- справочник выполняет поиск и фильтрацию;
- ИИ-наставник отвечает на пользовательские вопросы;
- прогресс сохраняется после перезапуска;
- документация и ТЗ приложены отдельными файлами.

## 12. Этапы выполнения

1. Анализ предметной области.
2. Определение требований.
3. Проектирование архитектуры.
4. Разработка интерфейса.
5. Разработка модулей уроков, практики, справочника и ИИ-наставника.
6. Реализация локального хранения.
7. Тестирование основных сценариев.
8. Подготовка документации.
9. Формирование итогового zip-архива.

## 13. Возможное дальнейшее развитие

- подключение реальной языковой модели;
- генерация индивидуальных заданий;
- проверка кода через sandbox-среду;
- личный кабинет с облачной синхронизацией;
- рейтинг и достижения;
- поддержка нескольких языков программирования.
